Redovisning av kursmoment i kursen databas

Kmom01

Att börja jobba med databaser, klienter och SQL är det något jag verkligen ser fram emot. Jag vill förstå hur data hanteras och struktureras på ett effektivt sätt, eftersom detta är en viktig del av alla moderna applikationer. Jag tror att jag kommer uppskatta att jobba med databaser och JavaScript. Jag hoppas att jag kommer att ta med mig mycket kunskaper från denna kurs som jag kommer ha stor nytt av i framtiden.

Jag har inte tidigare jobbat med databaser eller SQL men har någorlunda koll för konceptet kring hantering av data i tabeller. Tidigare har jag jobbat med programmeringsspråken python och C++ och där har jag jobbat med filer och enklare datahantering.

Arbetet med SQL i guiden tog lite tid innan jag började förstå vad jag håller på med. Men när jag väl lärde mig att skapa, läsa, uppdatera och ta bort data i en databas, blev det både roligt och lärorikt.

SQL skiljer sig från andra programmeringsspråk som Python och C++ exempelvis. Istället för att skriva deltaljerade instruktioner för varje steg för att uppnå ett resultat, som man gör i Python och c++, beskriver man i SQL vad man vill uppnå och låter databashanteraren ta hand om hur det ska genomföras. I SQL ställer man frågor direkt till databasen, medan andra språk kräver att man specificerar exakt hur varje uppgift ska lösas. Dessutom tycker jag att SQL har en lättare syntax jämfört med till exempel C++, som är mycket mer strikt och noggrann med detaljer.

Min TIL för detta kursmoment är att jag nu har lärt mig grunderna i SQL. Jag kan skriva kod för att hantera tabeller och hämta data på det sätt jag vill, både i terminalen och i verktyg som Workbench. Det känns som en bra grund för att bygga vidare på min förståelse för databaser.

Kmom02

Att använda både UNION och JOIN för att slå samman tabeller hjälper mig för att förstå hur man kan kombinera data på olika sätt. Arbetet går snabbare och man slipper göra samma sak flera gånger. Man kan joina tabeller med olika key för att kunna hämta relevant information från olika tabeller.

Nodejs var någonting nytt jag lärde mig och det verkar inte vara svårt att jobba i den miljön. Liknande har man gjort innan men node.js känns snabbare och mer effektivt. Man har hört att JavaScript bara är för frontend-utveckling, men Node.js visar verkligen hur kraftfullt det kan vara även på serversidan.

Asynkron programmering med async och await känns lite svårt men jag hoppas att jag kan lära mig mer om det. Att hämta data från en databas och vänta på att den ska returneras innan man visar informationen är lite kul och det visar hur asynkron programmering kan göra koden mer effektiv och lättläst. Till exempel i en väder applikation kan man använda async och await för att hämta väder data från en API och sedan uppdatera användargränssnittet när data är tillgänglig. Ett användbart verkty men det tar lite tid att vänja sig vid konceptet och förstå hur det fungerar under huven.

Att koppla ihop JavaScript och Node.js med en SQL-databas är ju viktigt för att bygga fullständiga applikationer. Det var inte så särskild komplicerad att koppla ihop så länge man följer instruktionerna och förstår på vilket sätt kopplingen ska ske.

Min TIL för detta kursmoment är att jag nu har lärt mig att använda olika paket för hantering av information inom terminalen. Att koppla ihop databas till node.js. Jag har fått lära mig mer om JavaScript och konceptet med async och await och mer SQl, att joina tabeller!

Kmom03
